# GL Assessment Research: Data Handling

Comprehensive research document covering how GL Assessment tests Data Handling (statistics, graphs, charts, tables, averages) in the 11+ maths paper. Built from web research across tutor resources, curriculum documents, practice paper analysis, and GL Assessment materials.

---

## Format

- **Paper:** GL 11+ Maths — 50 multiple-choice questions in 50 minutes
- **Answer format:** 5 options (A-E), mark one answer per question on a separate answer sheet
- **Data handling proportion:** Approximately 3-6 questions per paper (6-12% of the total). "Number" questions dominate (roughly 5x more than any other category), with Data, Shape/Space, Measure, and Algebra sharing the remainder
- **Question style:** Almost always presented as a stimulus (chart, graph, table, or dataset) followed by one or more questions requiring extraction, calculation, or comparison. Word problems referencing data are also common
- **Stimulus types:** Bar charts, pie charts, line graphs, tables, pictograms, tally charts, Venn diagrams, Carroll diagrams. The stimulus is printed in the question booklet; children read from it and select the correct multiple-choice answer
- **Time pressure:** At 1 minute per question, children must read the stimulus quickly, identify what is being asked, and calculate — speed and accuracy under pressure are essential

## Skills Tested & Category Weightings

| Skill Category | Estimated GL Weight | Core Operations |
|----------------|-------------------|-----------------|
| Reading values from charts/graphs | ~25-30% | Extract specific values from bar charts, line graphs, pictograms, tables |
| Calculating from data | ~25-30% | Find totals, differences, "how many more", fractions/percentages of totals |
| Averages (mean, median, mode, range) | ~15-20% | Calculate mean, find median (ordered data), identify mode, calculate range |
| Interpreting trends and comparisons | ~10-15% | Compare datasets, identify trends over time, draw conclusions |
| Pie chart interpretation | ~10-15% | Convert between fractions/percentages and sectors, calculate amounts from proportions |
| Critical data evaluation | ~5-10% | Identify what data does/doesn't show, spot misleading presentations |

### Specific Skills Breakdown

**Reading & Extraction:**
- Read exact values from bar charts (including reading between scale markings)
- Read pictograms with keys (e.g. one symbol = 5 items, half symbol = 2.5)
- Extract values from tables (rows and columns)
- Read line graphs at specific points
- Interpret tally charts (groups of 5)

**Calculation from Data:**
- Find totals by adding values across categories
- Find differences ("How many more children chose X than Y?")
- Calculate fractions of totals ("What fraction of the class chose football?")
- Calculate percentages from data
- Multi-step problems requiring two or more operations

**Averages & Statistical Measures:**
- Calculate the mean (sum divided by count)
- Find the median (middle value when ordered)
- Identify the mode (most frequent value)
- Calculate the range (highest minus lowest)
- "Backwards mean" — find a missing value given the mean and other values

**Pie Charts (cross-topic with fractions/percentages):**
- Read proportions from pie charts as fractions
- Convert between fractions, percentages, and degrees (360 degrees = whole)
- Calculate actual values from pie chart proportions given total
- Construct mental model of pie chart from data

**Data Comparison & Interpretation:**
- Compare values across categories or time periods
- Identify highest/lowest/most popular/least popular
- Read line graphs for trends (increasing, decreasing, steady)
- Conversion graphs (reading one axis from the other)
- Dual/grouped bar charts (comparing two datasets side by side)

## Difficulty Criteria

### D1 (Easy ~30%)

Questions at this level involve straightforward single-step reading or calculation from clearly presented data.

**Characteristics:**
- Direct value reading from a clearly labelled chart or table
- Values fall exactly on grid lines or scale markings (no estimation needed)
- Single operation required (read one value, or find a simple total)
- Familiar, everyday context (favourite colours, pets, sports)
- Small, manageable dataset (4-6 categories, single-digit or round numbers)
- Pictogram with simple key (1 symbol = 1 or 2 items)
- Bar chart with clearly labelled integer scale starting at 0
- "Which was the most popular?" or "How many children chose X?"

**Example question types:**
- "The bar chart shows children's favourite fruits. How many children chose bananas?" (read directly from bar)
- "Look at the pictogram. How many more sunny days were there than rainy days?" (count symbols, subtract)
- "What was the most popular colour?" (identify tallest bar)
- Table with 4 entries: "What is the total number of books read?"

**Distractors at D1:**
- Values from adjacent bars or wrong rows in tables
- Reading the wrong axis
- Simple arithmetic errors (off by one)

### D2 (Medium ~40%)

Questions at this level require either reading between scale markings, performing a two-step calculation, or applying a statistical concept.

**Characteristics:**
- Values may fall between grid lines (estimation/interpolation required)
- Two-step calculations: read two values then calculate difference, total, or fraction
- Requires understanding of scale intervals (e.g. each square = 5 units, so halfway = 2.5)
- Pictogram keys with multipliers (1 symbol = 10, half symbol = 5)
- Simple mean/median/mode/range calculations with small datasets (5-8 values)
- "What fraction of the total is X?" requiring both extraction and fraction calculation
- Comparing two categories or time periods
- Line graph requiring reading at a specific time point

**Example question types:**
- "The bar chart shows shop sales. The scale goes up in 10s. How many more books were sold on Wednesday than Monday?" (read between lines, subtract)
- "Find the mean of these test scores: 8, 6, 9, 5, 7" (sum = 35, divide by 5 = 7)
- "The pie chart shows 60 children's favourite sports. One quarter chose football. How many children chose football?" (60 x 1/4 = 15)
- "What is the range of temperatures shown in the table?"
- Conversion graph: "Use the graph to convert 5 miles to kilometres"

**Distractors at D2:**
- Misreading the scale interval (e.g. treating each line as 1 when it represents 5)
- Forgetting to divide by the count when finding the mean
- Confusing mean with median or mode
- Reading pictogram symbols without applying the key multiplier
- Fraction with wrong denominator (using one category instead of total)

### D3 (Hard ~30%)

Questions at this level require multi-step reasoning, working backwards from given information, combining data handling with other mathematical skills, or critical evaluation of data.

**Characteristics:**
- Multi-step problems requiring 3+ operations
- "Backwards mean" — given the mean and some values, find the missing value
- Pie chart questions requiring angle calculation or percentage-to-amount conversion with non-obvious fractions
- Combining information from two different representations (table + graph)
- Questions asking what the data does NOT show or whether a conclusion is valid
- Grouped or continuous data requiring interval reading
- Dual bar charts requiring comparison of differences between two datasets
- Time-series graphs requiring trend analysis or prediction
- Misleading data identification (truncated axes, cherry-picked data, averages without context)
- Large or complex datasets (10+ values, decimal numbers)

**Example question types:**
- "The mean score of 6 tests is 72. Five of the scores are 68, 75, 80, 63, and 70. What is the sixth score?" (72 x 6 = 432, subtract known scores, answer = 76)
- "A pie chart shows results for 240 students. The 'maths' sector has an angle of 90 degrees. How many students chose maths?" (90/360 = 1/4, 240 x 1/4 = 60)
- "The line graph shows temperature over 24 hours. Between which two consecutive hours was the greatest temperature drop?"
- "Tom says 'More people prefer tea than coffee because the tea bar is wider.' Is Tom correct? Explain why."
- "The table shows rainfall for 12 months. What percentage of months had more than 50mm rainfall?"
- Venn diagram with overlapping categories: "How many children play ONLY football?"

**Distractors at D3:**
- Multiplying instead of dividing for backwards mean (or vice versa)
- Forgetting to subtract known values when finding the missing number
- Incorrect angle-to-fraction conversion for pie charts
- Including overlapping sections when counting "only" in Venn diagrams
- Misidentifying the question (greatest increase vs greatest value)
- Calculation errors in multi-step chains that match plausible wrong answers

## Common Question Types

### Type 1: Direct Value Reading
"How many [category]?" or "What was the [value] for [category]?"
- Read a single value from a bar chart, table, pictogram, or line graph
- Appears at all difficulties depending on scale complexity
- Easiest when values align with grid lines; harder with estimation

### Type 2: Comparison Questions
"How many more X than Y?" or "What is the difference between X and Y?"
- Extract two values and calculate the difference
- Common with bar charts and tables
- GL may ask for greatest/smallest difference across multiple pairs

### Type 3: Total and Sum Questions
"How many children were surveyed altogether?" or "What is the total for all categories?"
- Add all values from a chart or table
- May require reading multiple bars/rows
- Sometimes combined with "What fraction of the total is [category]?"

### Type 4: Fraction/Percentage of Total
"What fraction of the children chose [category]?" or "What percentage [...]?"
- Requires finding the specific value AND the total
- Then expressing the answer as a simplified fraction or percentage
- Cross-topic with fractions and percentages skills

### Type 5: Average Calculations
"Find the mean/median/mode/range of this data"
- Mean: add all values, divide by count
- Median: order the values, find the middle one (or average of two middle values)
- Mode: identify the most frequent value
- Range: highest minus lowest
- GL may present data in a table or frequency chart rather than a simple list

### Type 6: Backwards Mean (Missing Value)
"The mean of 5 numbers is 12. Four of the numbers are 10, 14, 8, 15. Find the fifth number."
- Work backwards: mean x count = total sum, then subtract known values
- One of the most common "hard" data handling question types in 11+
- Tests genuine understanding of mean, not just the calculation formula

### Type 7: Pie Chart Interpretation
"The pie chart shows how 120 children travel to school. What fraction travel by bus?"
- Read the proportion as a fraction (quarter, third, half, etc.)
- Calculate actual numbers from proportions given the total
- May involve angle measurement (sector angle / 360)
- Often combined with percentage conversions

### Type 8: Line Graph Interpretation
"The graph shows temperature during the day. At what time was the temperature 15 degrees C?"
- Read values at specific points
- Identify trends (rising, falling, steady)
- Find greatest increase/decrease between consecutive points
- Interpolation between plotted points

### Type 9: Pictogram Interpretation
"Each symbol represents 10 books. How many books were read in March?"
- Apply the key (whole symbols and half symbols)
- Half symbols are a common source of error
- May need to work backwards: "Draw how many symbols for 35 books" (3.5 symbols)

### Type 10: Tally Chart / Frequency Table
"The tally chart shows..." or "Complete the frequency table"
- Count tally marks (groups of 5 with diagonal stroke)
- Convert between tally and frequency
- Often the first step before calculating totals or averages

### Type 11: Venn and Carroll Diagrams
"How many children play ONLY football?" or "How many are in the overlap?"
- Venn: understand union, intersection, and exclusive regions
- Carroll: sort by two criteria simultaneously
- Key trap: confusing "football" (all who play it) with "only football" (excluding overlap)

### Type 12: Conversion Graphs
"Use the graph to convert 8 kilometres to miles"
- Read across from one axis, up/down to the line, across to the other axis
- Tests graph reading precision
- May ask for approximate conversions at unmarked points

## Distractor Design

GL Assessment designs distractors (wrong answers) that catch specific, predictable errors. For data handling, the key distractor patterns are:

### 1. Scale Misreading Distractors
- **What GL does:** If the y-axis goes up in 5s (0, 5, 10, 15...) and a bar reaches halfway between 10 and 15, the correct answer is 12.5 or approximately 13. Distractors will include 12, 10, 15, and 3 (reading the number of grid squares instead of the value)
- **Why it works:** Children who don't check the scale interval assume each line = 1

### 2. Wrong Operation Distractors
- **What GL does:** For "How many more X than Y?", include options for X + Y (total instead of difference), X alone, Y alone, and X - Y with digits swapped
- **Why it works:** Children who misread the question or default to addition

### 3. Pictogram Key Error
- **What GL does:** If each symbol = 10 and there are 3.5 symbols, correct answer is 35. Distractors include 3.5 (forgot to multiply), 30 (ignored half symbol), 7 (halved instead of multiplied), 40 (rounded up)
- **Why it works:** Children forget to apply the key or misinterpret half symbols

### 4. Mean Calculation Errors
- **What GL does:** For mean of {4, 6, 8, 10, 12}, correct = 8. Distractors include 40 (sum only, forgot to divide), 12 (highest value), 8 (median, coincidentally same here but GL would use datasets where mean and median differ), 6 (divided by wrong count)
- **Why it works:** Each distractor catches a specific procedural error

### 5. Fraction/Percentage Denominator Error
- **What GL does:** For "What fraction chose football?" where 12 out of 48 children chose it, correct = 1/4. Distractors include 12/36 (using remainder instead of total), 1/12 (inverted), 12/48 (unsimplified, if options require simplified form), 1/3
- **Why it works:** Children divide by the wrong number or forget to simplify

### 6. Pie Chart Proportion Errors
- **What GL does:** If a pie chart sector is one-third and the total is 90, correct = 30. Distractors include 60 (two-thirds), 45 (half), 33 (confused percentage with actual value), 3 (divided total by incorrect fraction)
- **Why it works:** Fraction-of-an-amount errors compound with pie chart reading errors

### 7. Venn Diagram Region Confusion
- **What GL does:** For "How many play ONLY tennis?", include the overlap count, the total tennis count, and the "only football" count
- **Why it works:** Children confuse "tennis" (whole circle) with "only tennis" (excluding overlap)

### 8. Median Ordering Error
- **What GL does:** Present data in non-sequential order. Correct median requires ordering first. Distractor = middle value of unordered list
- **Why it works:** Children skip the ordering step and pick the middle position from the given list

### 9. Range vs Highest Value
- **What GL does:** Include the highest value as a distractor for range questions
- **Why it works:** Children forget that range = highest MINUS lowest, not just the highest

### 10. Off-by-One in Counting
- **What GL does:** Include values that are 1 more or 1 less than the correct answer
- **Why it works:** Miscounting tally marks, bars, or data points — especially in larger datasets

## Common GL Traps

1. **Scale interval trap:** The y-axis does not go up in 1s. Children must check the interval. GL commonly uses intervals of 2, 5, 10, 20, 50, or 100. A bar that reaches the 3rd line on a scale going up in 5s represents 15, not 3.

2. **Pictogram half-symbol trap:** A pictogram shows 2.5 symbols where each symbol = 10. Many children answer 2.5, 25, or 20 instead of the correct 25. The half-symbol is the single most common pictogram error.

3. **Horizontal vs vertical axis confusion:** Children mix up which axis shows what. GL sometimes uses horizontal bar charts to exploit this further.

4. **"How many more" ≠ "How many":** Children read "How many more did X get than Y?" and answer with just X's value instead of the difference.

5. **Mean vs total trap:** Children calculate the total but forget the final division step, especially under time pressure. The total is always offered as a distractor.

6. **Backwards mean misdirection:** When given mean = 8 for 5 values, children must calculate total = 40 first. Common error: subtracting the mean from each known value instead of from the total.

7. **Unordered median trap:** Data is deliberately presented out of order. The child who picks the "middle number" without sorting first gets the wrong answer — and that wrong answer is always an option.

8. **Mode with equal frequencies:** GL may include datasets where two values appear the same number of times, or where every value appears once (no mode). This tests whether children truly understand the concept.

9. **Pie chart estimation trap:** A sector that looks like "about a quarter" might actually be 30% or 20%. GL tests whether children can calculate precisely rather than estimate visually.

10. **Missing the question focus:** A graph shows data for 5 days. The question asks about "Tuesday and Wednesday combined." Children who answer about just one day, or about all 5 days, fall into this trap.

11. **Tally mark miscounting:** The fifth tally mark crosses the other four diagonally. Children sometimes count groups as 4 instead of 5, or miscount partial groups.

12. **Conversion graph direction error:** Reading from the wrong axis (converting pounds to kilograms instead of kilograms to pounds).

13. **Dual bar chart confusion:** When two bars sit side by side for each category, children may compare the wrong bars or add when they should subtract.

14. **"Not shown" trap (D3):** The data shows X but the question asks about Y. For example, the chart shows "number of children" but the question asks about "percentage" — requiring an extra calculation step.

## Key Concepts Reference

### Averages

| Measure | Definition | How to Calculate | Common Error |
|---------|-----------|-----------------|--------------|
| **Mean** | The "average" — sum of values divided by count | Add all values, divide by how many there are | Forgetting to divide; dividing by wrong number |
| **Median** | The middle value when data is ordered | Put values in order smallest to largest, find the middle one. For even count: average the two middle values | Not ordering first; wrong position for even-count data |
| **Mode** | The most common value | Find the value that appears most often | Confusing with median; claiming no mode when one exists; giving frequency instead of the value |
| **Range** | The spread of the data | Highest value minus lowest value | Giving the highest value only; giving the difference between adjacent values |

### Backwards Mean Formula
If the mean of *n* numbers is *m*, then:
- Total sum = n x m
- Missing value = (n x m) - sum of known values

**Example:** Mean of 5 numbers is 12. Four numbers are 10, 14, 8, 15.
- Total = 5 x 12 = 60
- Known sum = 10 + 14 + 8 + 15 = 47
- Missing number = 60 - 47 = **13**

### Pie Chart Calculations

| Given | Find | Formula |
|-------|------|---------|
| Fraction of total | Angle | Fraction x 360 |
| Angle | Fraction | Angle / 360 |
| Fraction + total count | Actual number | Fraction x total |
| Angle + total count | Actual number | (Angle / 360) x total |
| Percentage + total count | Actual number | (Percentage / 100) x total |

### Data Representation Types

| Representation | Best For | Key Feature | GL Trap |
|---------------|----------|-------------|---------|
| **Bar chart** | Comparing categories | Height of bar = value | Scale doesn't start at 0; unequal intervals |
| **Pie chart** | Showing proportions | Sector angle = proportion of 360 degrees | Estimating vs calculating; fraction errors |
| **Line graph** | Showing change over time | Points connected by lines | Reading between points; confusing axes |
| **Pictogram** | Visual counts | Symbols represent quantities via key | Half symbols; forgetting to apply key |
| **Table** | Exact values, multiple categories | Rows and columns | Reading wrong row/column; totalling errors |
| **Tally chart** | Recording raw data | Groups of 5 strokes | Miscounting the diagonal fifth mark |
| **Venn diagram** | Overlapping categories | Circles showing intersection | "Only X" vs "X including overlap" |
| **Carroll diagram** | Sorting by two criteria | 2x2 grid | Placing items in wrong cell |
| **Conversion graph** | Converting between units | Straight line through origin | Reading from wrong axis |
| **Dual bar chart** | Comparing two datasets | Side-by-side bars per category | Comparing wrong bars; misidentifying legend |

## Key Vocabulary

Children must understand these terms in exam context:

### Data Collection & Organisation
- **Data** — facts or pieces of information collected about something
- **Tally** — a mark used to record one count (every 5th mark is diagonal)
- **Frequency** — how many times a value or category occurs
- **Survey** — collecting data by asking people questions
- **Category** — a group or class that data is sorted into

### Charts & Graphs
- **Bar chart** (bar graph) — uses rectangular bars to show quantities
- **Pie chart** — circular chart divided into sectors showing proportions
- **Line graph** — data points connected by lines, usually showing change over time
- **Pictogram** — chart using pictures/symbols to represent data, with a key
- **Key** (legend) — explains what each symbol represents in a pictogram or chart
- **Axis** (plural: axes) — the horizontal (x-axis) and vertical (y-axis) lines on a graph
- **Scale** — the numbered intervals along an axis
- **Sector** — a "slice" of a pie chart
- **Horizontal** — going across (left to right)
- **Vertical** — going up and down

### Statistical Measures
- **Mean** — add all values, divide by how many (the "average")
- **Median** — the middle value when data is in order
- **Mode** — the value that appears most often
- **Range** — difference between highest and lowest values
- **Average** — usually refers to the mean, but can mean any measure of central tendency
- **Total** — the sum of all values

### Comparison & Interpretation
- **Greatest / largest / highest / maximum** — the biggest value
- **Least / smallest / lowest / minimum** — the smallest value
- **Difference** — the result of subtracting one value from another
- **Increase / decrease** — going up or going down
- **Trend** — a general pattern or direction in data over time
- **Proportion** — a part compared to the whole
- **Estimate** — an educated guess of a value (especially reading between scale lines)

### Venn/Carroll Diagram Terms
- **Set** — a collection of items with a shared property
- **Intersection** (overlap) — items belonging to both sets
- **Union** — all items in either or both sets
- **Only** — items in one set but NOT the other (excludes the overlap)
- **Neither** — items outside all sets

## National Curriculum Alignment

### Year 5 Statistics Objectives
- Solve comparison, sum and difference problems using information presented in a line graph
- Complete, read and interpret information in tables, including timetables

### Year 6 Statistics Objectives
- Interpret and construct pie charts and line graphs and use these to solve problems
- Calculate and interpret the mean as an average

### Notes on Curriculum vs 11+ Scope
- **Median, mode and range** are not explicitly required in the Year 5-6 National Curriculum but ARE commonly tested in GL 11+ papers. The curriculum focuses on mean only, but GL expects familiarity with all four measures
- **Probability** (likelihood, chance, the probability scale from 0 to 1) is part of the KS2 curriculum and occasionally appears in GL papers as a data-adjacent topic. Children should understand: impossible, unlikely, even chance, likely, certain — and express probability as fractions, decimals, or percentages
- **Venn diagrams and Carroll diagrams** are taught from Year 3-4 and occasionally appear at 11+ level, though less frequently than bar charts and pie charts
- **Conversion graphs** are a Year 6 measurement topic but often appear in the data handling section of 11+ papers
- **Grouped data and frequency tables** are KS2 objectives (Year 6: "construct and interpret frequency tables, bar charts with grouped discrete data")
- **Timetables** (bus/train schedules) are explicitly in the Year 5 curriculum and commonly tested in 11+

### What GL Tests Beyond Strict Curriculum
GL Assessment papers can include questions at or slightly above Year 6 level. For data handling, this means:
- More complex multi-step problems than standard SATs questions
- Backwards mean questions (find missing value from the mean)
- Combined data representation questions (read from one chart, calculate, compare to another)
- Misleading data questions (critical thinking about data presentation)
- Larger datasets and less friendly numbers (decimals in means, non-obvious fractions in pie charts)

## Teaching Notes

### Progression Sequence for Data Handling
Follow the evidence-based "Read, Interpret, Create" progression:

1. **Read the data** — can the child accurately read values from charts, graphs, and tables?
2. **Read between the data** — can they find relationships, make comparisons, calculate totals and differences?
3. **Read beyond the data** — can they draw conclusions, identify trends, evaluate whether claims are supported?

### Priority Skills for GL Preparation

**Must-know (appears on almost every paper):**
- Reading bar charts with non-unit scales (intervals of 2, 5, 10)
- Calculating mean from a small dataset
- Finding totals and differences from tables
- Reading pictograms with keys (including half symbols)
- Simple pie chart reading (quarter, half, third)

**Should-know (appears frequently):**
- Median, mode, and range
- Pie chart calculations with given totals
- Line graph reading at specific points
- Fraction-of-total questions from any chart type
- Tally chart interpretation

**Could-appear (occasional, usually D3):**
- Backwards mean (missing value)
- Conversion graphs
- Venn diagram problems
- Dual bar chart comparisons
- Misleading data / critical evaluation
- Grouped frequency tables
- Combining information from two representations

### Common Mistakes to Drill

These are the errors that lose marks most often. Practise each specifically:

1. **Not checking the scale interval** — always look at the numbers on both axes before reading any values
2. **Not ordering data before finding median** — make it automatic: "median = order first"
3. **Confusing mean, median, mode** — use mnemonics: "Mean = add and share equally, Median = middle, Mode = most"
4. **Forgetting the pictogram key** — always read the key first, then count symbols and multiply
5. **Giving the total when asked for the mean** — check: "Did I divide?"
6. **Giving the highest value when asked for the range** — range = highest MINUS lowest, always a subtraction
7. **Reading the wrong axis on graphs** — practice: point to the x-axis label, then the y-axis label, before answering
8. **Answering "how many" when asked "how many more"** — underline the word "more" in the question
9. **Venn diagram "only" errors** — draw the regions and label what each part means before answering

### Time Management Strategy
Data handling questions often have a stimulus (chart/graph) that takes time to absorb. Strategy:
- Spend 5-10 seconds scanning the stimulus before reading the question
- Identify what the axes/key represent
- Then read the question and go directly to the relevant part of the data
- If a question seems to need 3+ steps, consider flagging it and coming back — it might be a D3 question worth 1 mark just like an easy D1 question

### Visual Component Recommendations for App
Based on GL question formats, the most valuable visual components for quiz questions are:
- Bar charts (vertical and horizontal, various scales)
- Pie charts (with sector labels as fractions, percentages, or degrees)
- Line graphs (time series, conversion graphs)
- Pictograms (with keys showing whole and half symbols)
- Tables (simple and two-way)
- Venn diagrams (two-circle, with numbers in each region)
- Tally charts (for frequency questions)
