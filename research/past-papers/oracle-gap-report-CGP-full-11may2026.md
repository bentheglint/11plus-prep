## MATHS GAPS

### NEW Gaps (not in previous report)

#### 1. Misleading charts / chart criticism
- **Skill:** Identify why a graph or pie chart is misleading (y-axis doesn't start at 0; unequal intervals; section sizes; misleading visual scaling)
- **CGP frequency:** ~3 questions per 37-test run (10-min Test 4 Q8, Test 11 Q5, Test 25 Q5)
- **Maps to topic:** `datahandling`
- **Severity:** Important
- **Recommended fix:** Add 12–15 questions (across D2/D3) covering: y-axis not starting at zero; unequal scale intervals; misleading section/area proportions; insufficient sample size critique. Distractors should include plausible-but-wrong reasons (e.g. "not enough flavours" when the real issue is scale).

#### 2. Conversion of units — selecting "most suitable unit"
- **Skill:** Choose appropriate unit for a real-world measurement (depth of well, distance between cities, capacity of fish tank)
- **CGP frequency:** ~3 questions per 37-test run (Test 8 Q2, Test 15 Q5, Test 18 Q2)
- **Maps to topic:** `placevalue` (or `decimals` for some unit-conversion problems)
- **Severity:** Important
- **Recommended fix:** Add 10 questions covering: selecting the correct unit (mm/cm/m/km, ml/l, g/kg/tonnes) for sensibly-scaled real items. Include the trap unit (m² for depth, m³ for length).

#### 3. Multi-unit conversion / arithmetic across mixed units
- **Skill:** Mixed-unit arithmetic — adding/subtracting masses, capacities, lengths given in different units (g + kg, ml + l, m + cm, mm + cm)
- **CGP frequency:** ~10–12 questions per 37-test run (Test 1 Q4, Test 1 Q6, Test 3 Q4, Test 5 Q4, Test 7 Q3, Test 11 Q6, Test 12 Q3, Test 13 Q5, Test 14 Q8, Test 18 Q3, Test 20 Q1)
- **Maps to topic:** `decimals` (and `placevalue`)
- **Severity:** Critical (high GL frequency — appears in nearly every test)
- **Recommended fix:** Add 20–25 questions where the child must first convert units to a common base, then perform arithmetic. Mix of D2/D3. Cover g↔kg, ml↔l, mm↔cm↔m, cm↔m↔km. Include real-world wrappers (recipes, bottles, distances).

#### 4. Multi-step data interpretation (combination / ranking)
- **Skill:** Reading bar charts/line graphs where the answer is NOT the obvious tallest bar (e.g. "which 3 players' totals equal X"; "next longest after Cooper")
- **CGP frequency:** ~5 per 37-test run (Test 9 Q6, Test 10 Q7, Test 13 Q2)
- **Maps to topic:** `datahandling`
- **Severity:** Minor
- **Recommended fix:** Add 6–8 questions requiring two-step reasoning from a chart (combine multiple bars to match a target; find Nth ranked item from a table).

#### 5. Sequence pattern → nth term expression
- **Skill:** Given a visual pattern (dots/squares/shapes), find the nth term as an expression OR identify how many in the nth pattern
- **CGP frequency:** ~5 per 37-test run (Test 3 Q10, Test 10 Q6, Test 14 Q10, Test 19 Q10, Test 27 Q10, GL Mock Q10)
- **Maps to topic:** `sequences` (and `algebra` overlap)
- **Severity:** Important
- **Recommended fix:** Add 12–15 questions: visual pattern → expression (e.g. perimeter = 3n+1); given a formula → find term N; given a term value → find which pattern. Confirms the previous report's "nth term / algebraic expressions for sequences" as a real gap.

#### 6. Triangular numbers / square numbers as sequences
- **Skill:** Recognise quadratic-like patterns (1, 3, 6, 10, 15 …) and multiply-and-add rules
- **CGP frequency:** ~4 per 37-test run (Test 10 Q6, Test 1 Q8, GL Mock Q10)
- **Maps to topic:** `sequences`
- **Severity:** Important
- **Recommended fix:** Add 10 questions covering triangular numbers, square numbers, sequences with rule "multiply by N then add M" (e.g. ×4+1 giving 1, 5, 21, 85, 341…).

#### 7. "Thinks of a number" reverse-operation puzzles (multi-step)
- **Skill:** Reverse a sequence of operations to find the starting number (Cally doubles, adds 4, doubles again, gets 44 → start = 9)
- **CGP frequency:** ~6 per 37-test run (Test 1 Q9, Test 7 Q4, Test 16 Q7, Test 19 Q3, GL Mock Q23)
- **Maps to topic:** `algebra`
- **Severity:** Important

#### 8. Algebraic expression to model a real-world rule
- **Skill:** Write an expression like "30 − 2S" or "4 + 0.8d" from a real-world story (flat rate + per-mile, jars minus consumed, stacked pots with overlap)
- **CGP frequency:** ~8 per 37-test run (Test 6 Q10, Test 8 Q10, Test 9 Q10, Test 10 Q10, Test 18 Q10, Test 20 Q8, Test 24 Q10, Test 27 Q10, GL Mock Q37, Q50)

- **Recommended fix:** Add 18–22 questions where the child must BUILD an expression rather than evaluate one. Cover formula machines, flat-rate + variable cost, stacking with overlap, age problems. The app's algebra likely focuses on solving equations; constructing them is different.

### Gaps / Coverage Concerns

#### V1. Balance-equation arithmetic (missing number in equations)
- **Gap name:** Solve `15 + 4 = 29 − ( )`, `40 − 30 + 20 = 3 × ( )`
- **CGP frequency:** ~12 per 37-test run (appears as dedicated 4-Q sections in 7+ tests)
- **App topic:** No mapping — not `letterSums`, not `numberSeries`
- **Issue:** Up to 16% of some tests. Requires BODMAS application. Not in app at all.
- **Severity:** Critical
- **Recommended fix:** Add 40 questions as a new sub-type (could live under `letterSums` or as `equationSolving`). Cover: simple balance, BODMAS, negative results, mixed operations.

#### V2. Word-pattern transformations
- **Gap name:** `frog (rode) dent` → take letters 3-4 of word 1 + letters 1-2 of word 3; `seam ( ) pity` → ?
- **CGP frequency:** ~16 per 37-test run (dedicated sections in 8+ tests)
- **App topic:** `wordCodeAnalogies`
- **Issue:** Structural letter-pattern extraction — different from cipher codes. Need to confirm app coverage.
- **Severity:** Important — audit needed
- **Recommended fix:** Audit `wordCodeAnalogies`. If structural transformations (take-letters-from-each-side) <30 questions, add 25–30 more.

#### V3. Three-letter-word INSIDE capitals (sentence context)
- **Gap name:** `He SWALED the custard` → insert "LOW" to make SWALLOWED
- **CGP frequency:** ~18 per 37-test run (dedicated 4-Q sections in 7 tests)
- **App topic:** `missingLettersWords`
- **Issue:** CGP inserts the 3-letter word INSIDE the capitalised word (between two halves). App has CH___ER format (end of word). These are different tasks.
- **Severity:** Important
- **Recommended fix:** Add 40 questions in format `SWALED` (insert 3 letters inside to complete the word, with sentence context).

#### V4. Multi-attribute logic problems ("must be true / cannot be true")
- **Gap name:** 5-person logic puzzles, 4–6 conditions, "which MUST be true?"
- **CGP frequency:** ~10 per 37-test run (9+ tests + GL Mock Q24, 25, 43, 44)
- **App topic:** `logicAndLanguage`
- **Issue:** App appears to have simpler 3-person ranking logic. CGP requires building a mental matrix and distinguishing must/could/cannot.
- **Severity:** Critical
- **Recommended fix:** Add 30–35 questions at D2/D3 with 5 people × 4+ conditions. Half "must be true," half "cannot be true."

#### V5. Middle-number analogies
- **Gap name:** `3 (18) 6 / 4 (20) 5 / 2 (?) 6`
- **CGP frequency:** ~12 per 37-test run (dedicated 4-Q sections in 7 tests)
- **App topic:** `numberSeries`
- **Issue:** Entirely missing from app. Rules range from simple (multiply, add) to complex composites.
- **Severity:** Critical
- **Recommended fix:** Add 40 questions as a new sub-type within `numberSeries`. Rules to cover: multiply outers, add outers, subtract outers, ½ sum, sum × something, larger ÷ smaller. Mix D1/D2/D3.

#### V6. Compound-word format (3-from-3 brackets)
- **Gap name:** `(cot miss comb) (shun tun at)` — pick one from each set
- **CGP frequency:** ~24 per 37-test run
- **App topic:** `compoundWords`
- **Issue:** Format matches but CGP consistently uses 3 words per bracket. App may use 4 or 5.
- **Severity:** Minor — confirm format match
- **Recommended fix:** Audit `compoundWords` option counts. Standardise to 3-from-each-bracket if needed.

#### V7. Synonyms — polyseme format (both-meaning match)
- **Gap name:** `(role character) (separate divide)` → underline word from list that matches BOTH meanings (e.g. "part")
- **CGP frequency:** ~12 per 37-test run (8 tests + GL Mock)
- **App topic:** `synonyms`
- **Issue:** App uses pick-one-from-each-of-two-bracket-sets (pure synonymy). This CGP format requires finding a polyseme — a word satisfying two distinct meanings simultaneously. Different cognitive task.
- **Severity:** Important — genuinely different question type
- **Recommended fix:** Add 30 polyseme-synonym questions (single answer from word list satisfying two distinct meaning groups).

#### V8. Letter pair series (different-rule-per-letter variants)
- **Gap name:** `BB CC EE HH LL (?)` where both letters follow the same stepping rule
- **CGP frequency:** ~16 per 37-test run
- **App topic:** `letterPairSeries`
- **Issue:** Likely covered but confirm app has different-rule-per-letter variants (first letter +1, second letter −2).
- **Severity:** Minor — audit
- **Recommended fix:** If different-rule variants <50 questions, add 20 more.

## Key Patterns Observed

1. **Maths: mixed units appear in virtually every CGP 10-min test** — at least one g/kg, ml/l, or mm/cm/m conversion-then-arithmetic problem per test.
2. **VR: CGP open-answer format ≠ MCQ format** — most question types translate fine to MCQ, but for equation-solving and three-letter-word-inside-capitals, plausible MCQ distractors need deliberate design.
3. **CGP VR is strictly 4 sections × 4 questions** — each test cycles through 4 question types. Worth knowing for pacing/sprint design.
4. **Logic problems are consistently 5-person, 4–6 conditions** — this is the GL standard. Anything simpler is underselling the difficulty.
5. **"Use this fact to find …" appears in both Maths and VR** — shared underlying skill of structural transfer.
