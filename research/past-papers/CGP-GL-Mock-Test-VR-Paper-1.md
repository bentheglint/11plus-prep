## Potential Coverage Gaps — For Oracle Review

### 1. Middle-number analogies (Q66–70) — format distinction from number series
The format `3(18)6 :: 4(20)5 :: 2(?)6` requires finding the rule connecting two outer numbers to produce the middle number. This is **mathematically and cognitively different** from a linear number series (Q18-23). The app's `numberSeries` topic may not explicitly cover this sub-type. Rules can include: multiply, add, subtract, or more complex combinations of the outer numbers.

### 2. Letter codes — two distinct sub-types in one topic
Q1-5 (alphabet position shift: AC:CE::FH:?) and Q36-42 (cipher: HELP=KHOS, what is VWDUW?) are both in the `letterCodes` topic but require completely different skills. Worth checking the app teaches both sub-types explicitly, not just one.

### 3. Logic problems with multiple conditions (Q24–25, Q43–44)
These require constructing a mental matrix from 5 people × multiple conditions, then identifying the one statement that MUST be true (vs could be true). This is harder than simple two-condition logic. 4 questions in this paper — worth checking app depth.

### 4. Compound words (Q26–30) — photo quality note
Several compound word pairs were hard to read precisely due to photograph angle. The Oracle should be aware that more creative compound combinations may appear than the standard pairs already in the app bank.

### 5. Odd two out — 2-from-5 format confirmed
This paper confirms the GL format is always "find the TWO unrelated words from five" (not one). If any app questions ask for only one, that's a format error.

### 6. Number-word codes — shared code set across sub-questions
Q50-52 share one code set; Q53-55 share another. Both sets require working out the full code before answering 3 sub-questions. If the app presents each number-word code question in isolation (with its own code given), it may not replicate the exam experience of building the code from partial information.
